## What is a Tier0 App

A Tier0 application has the following features:
- It is a **Docker Container**.
- It is generated from the conversation you have with the **App Builder**, with no programming required.
- It feeds on industrial data already in the **UNS**, and can *write back* to the **UNS**.
- It can use **Event Flow** to implement logic.

## How to Deploy a Tier0 App

### Building a Tier0 App from Scratch
<Steps>
1. Log in to Tier0, go to **App** > **Projects** and create a project for the application.
2. Create an application inside the project, and you will be redirected to the application building page.
3. Write the application requirements in the dialog.

    :::tip[What to include in the prompt]
    Make sure you state the following information in the prompt:
    - Use the UNS model *[uns_model]* as the data source and *[uns_model]* as the write-back destination.
    - (optional) Create UNS models and Source Flow flows to connect data sources.
    - Create flows in Event Flow to trigger anomaly alarms/low health scores/other events.
    :::

4. Start building the application. After previewing it, ask the LLM to make any changes you need.
5. Click **Deploy** to finish the application building.
</Steps>

### Building a Tier0 App from Templates
:::note[Why do templates exist]
- Existing templates are applications that are commonly used in today's industry.
- Building applications from templates prevents you from doing similar work from scratch.
:::
<Steps>
1. In Tier0, go to **Library** and select a template as needed.
2. Use the template as the application base and edit it in **Builder** through natural conversation with the LLM.
3. Deploy the application.
</Steps>

### Deploying App on Tier0 Enterprise
<Steps>
1. Find the application inside your project.
2. Access it, click the **...** icon of the **Active** version to **Download ZIP**.
3. Log in to Tier0 Enterprise, go to **Project** and create a project for the application.
4. Click **Import App** and upload the ZIP package.
</Steps>
