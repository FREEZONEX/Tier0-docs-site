Tier0 ships in three editions:
- **Edge** - Open-source deployment, runs locally with Docker.
- **Cloud** - Managed SaaS, no infrastructure required.
- **Enterprise** - Private deployment for enterprise environments.

See [Choosing the Best Version](../choosing-version/) to find the edition that best fits your requirements.


## Installing Tier0

<Tabs syncKey="edition">
	<TabItem label="Cloud">

		Apply for a trial account before exploring Tier0 Cloud.
		<Steps>
		1. Register your account at [Register Tier0](https://tier0.dev/signup). 
		2. Enter the verification code sent to your email on the popup window.
		3. You will be redirected to the **Login** page. Log in with the registered account.
		4. Enter related information, click **Continue**.
		5. Select **Create a New Workspace**.
		6. Create a workspace and start your trial.
	</Steps>
	
	</TabItem>
	<TabItem label="Edge">

	#### Requirements
	- Minimum 4-core CPU, 8 GB RAM, 100 GB disk
	- Ubuntu 24.04 or Windows 10/11 with Docker and Git
	#### Operations
	<Steps>
	1. Clone the Tier0-Edge repository from GitHub and access the project.

		```bash
		git clone https://github.com/FREEZONEX/Tier0-Edge
		cd Tier0-Edge/deploy
		```

	2. Configure the parameters listed in the table below in `.env.default` according to your environment.
	
		```bash
		vi .env.default
		:wq!
		```
	3. Run the installation script.

		```bash
		bash bin/install.sh
		```

	</Steps>

:::note[Parameters to be configured]

| Field | Type | Description |
|------|------|-------------|
| VOLUMES_PATH | string | Storage path for project data |
| ENTRANCE_DOMAIN/ENTRANCE_PORT | string | Tier0 access endpoint (IP/domain + port) |
| OS_PLATFORM_TYPE | enum | OS where Tier0 is installed. Can be **linux** or **windows** |
| LANGUAGE | enum | zh-CN / en-US |

:::
		

	</TabItem>
	<TabItem label="Enterprise">
#### Requirements
-  Standard Deployment
	- Minimum: 4 VMs
	- Required baseline per VM types:
		- App: 8C / 32G / 100G
		- Database: 4C / 8G / 500G
		- Witness: 2C / 4G / 40G
-  High Availability Deployment
	- Minimum: 6 VMs
	- Required baseline per VM types:
		- App (Swarm): 8C / 32G / 100G x 3
		- Database: 4C / 8G / 500G x 2
		- Witness: 2C / 4G / 40G

:::tip[Hardware requirements may vary]
Enterprise hardware requirements vary by deployment scenario.
:::

:::note
Detailed deployment information is available in the [High Availability Deployment Plan](../../reference/high-availability-deployment/).
:::
#### Operations
<Steps>
1. Get the package from your vendor, and verify it.
		```bash
		sha256sum -c tier0-enterprise-offline-standard-YYYYmmddTHHMMSS.tar.gz.sha256
		```
2. Decompress the package.
		```bash
		tar -xzf tier0-enterprise-offline-standard-YYYYmmddTHHMMSS.tar.gz
		cd tier0-enterprise-offline-standard-YYYYmmddTHHMMSS
		```
3. Edit `install.env` based on the deployment details, then start the deployment.
		```bash
		vi install.env
		bash install.sh
		```
</Steps>

#### Common Maintenance Operations
- Check the deployment environment without installing Enterprise.
		```bash
		./install.sh --check-only
		```
- Clear all nodes after a failed deployment.
		```bash
		./full-cleanup.sh --env install.env --yes
		```
		> This command wipes all Tier0 runtime data and database backups while keeping the VMs, operating system, and installed Docker software.
- A deployment report is generated after installation.
		```text
		runtime/install-reports/<timestamp>/deployment-report.md
		```
		> Do not deliver the project if the report contains any **FAIL** item. Contact support first.
