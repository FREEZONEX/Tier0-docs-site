本节以 OPC UA、Modbus 和 API 等常见数据类型为例，展示如何通过 **Source Flow** 将数据源连接到 **UNS**。

## OPC UA

### 配置数据流

<Steps>
1. Log in to Tier0, go to **Flow** and create a new **Source Flow**.
2. Add **inject**, **OPC UA Client** and **mqtt out** nodes in the flow.
3. Connect them by order and double-click each node to configure the details.
</Steps>
- **inject**
    <Steps>
    1. Click **add** at the lower-left corner to add an entry of property.
    2. Set the properties to be the following items:

        ```json
        {
          "msg.payload": "[multiple data tags]",
          "msg.topic": "multiple"
        }
        ```
    </Steps>
- **OPC UA Client**
    ```json
    "Endpoint": "<OPC UA server address and port>", //e.g. opc.tcp://127.0.0.1:4850
    "Action": "SUBSCRIBE",
    "Interval": "<data collecting interval>" //e.g. 10 seconds
    ```
- **mqtt out**
    ```json
    "Server": "<embedded UNS broker>", //same name as the flow
    "Topic": "<UNS model>",
    ```
<Steps>
4. Deploy and trigger the flow.
</Steps>

## Modbus
### 配置数据流
<Steps>
1. In the flow, add **modbus-read** and **mqtt out** nodes.
2. Double-click the nodes, and configure the required information.
    - **modbus-read**
        - **Settings**
          - **FC**: Function code that specifies the Modbus action, e.g., FC 3 = Read Holding Registers.
          - **Address**: The starting register address to read from (usually zero-based).
          - **Quantity**:  The number of consecutive registers to read.
          - **Poll Rate**: How often the node polls the Modbus device (e.g., every 10 seconds).
          - **Server**: Reference to a configured Modbus server (IP, port, protocol, etc.).
        - **Server**
          - **Host**: Modbus server IP.
          - **Port**: Modbus server port.
    - **mqtt out**
        ```json
        "Server": "<embedded UNS broker>", //same name as the flow
        "Topic": "<UNS model>",
        ```
3. Deploy and trigger the flow.
</Steps>

## API
### 配置数据流
<Steps>
1. In the flow, add **inject**, **http request** and **mqtt out** nodes.
2. Double-click the nodes, and configure the required information.
    - **http request**
        ```json
        "Method": "<API method>",
        "URL": "<API URL>",
        "Return": "a parsed JSON object",
        "Headers": "<API headers>"
        ```
    - **mqtt out**
        ```json
        "Server": "<embedded UNS broker>", //same name as the flow
        "Topic": "<UNS model>",
        ```
3. Deploy and trigger the flow.
</Steps>
