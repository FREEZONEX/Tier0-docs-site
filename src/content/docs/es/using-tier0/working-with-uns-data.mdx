Tier0 te permite operar sobre UNS mediante MQTT y APIs.

## MQTT
Tier0 permite que MQTT clients se conecten al broker UNS con credentials.
:::note
Puedes usar MQTT clients como **MQTTX**, **MQTT Explorer** y **Node-RED** para conectarte al broker y publicar/suscribirte a messages.
:::
### Generar MQTT Credentials
En la página **Edge** de Tier0, crea una nueva credential y guárdala.

### Conectar MQTT Clients

:::tip
Los siguientes clients se usan como ejemplos para mostrar cómo usar las credentials para conectarse al broker UNS.
:::

<Tabs syncKey="mqtt-client">
  <TabItem label="MQTTX">

1. Install and open <a href="https://mqttx.app/" target="_blank" rel="noopener noreferrer">MQTTX</a>.
2. Add a connection with the information of the Tier0 UNS broker, and the generated credentials.
    ```json
    {
      "Host": "mqtt.tier0.dev",
      "Port": 8883,
      "Client ID": "<generated client ID>",
      "Username": "<generated username>",
      "Password": "<generated password>"
    }
    ``` 

  </TabItem>
  <TabItem label="MQTT Explorer">

1. Install and open <a href="https://mqtt-explorer.com/" target="_blank" rel="noopener noreferrer">MQTT Explorer</a>.
2. Add a new connection with the broker information and the generated credentials.
    ```json
    {
      "Protocol": "mqtt://",
      "Host": "mqtt.tier0.dev",
      "Port": 8883,
      "Username": "<generated username>",
      "Password": "<generated password>"
    }
    ``` 
3. Change the client ID to the credential ID in **ADVANCED** before connecting.

  </TabItem>
  <TabItem label="Node-RED">
      - Use **Event Flow** - Data processing module in Tier0 based on Node-RED.
          1. Create a flow in **Event Flow**, and start it with an `mqtt in` node.
          2. Select the UNS broker as **Server** (same name as the flow) and set **Topic** to a UNS model in the node.
      - Use **Separate Node-RED Instance**
          1. Install and open the <a href="https://nodered.org/docs/getting-started/local" target="_blank" rel="noopener noreferrer">Node-RED</a> instance.
          2. Add a server in the `mqtt in` node with the following information.
          ```json
          {
            {/* under Connection */}
            "Server": "mqtt://mqtt.tier0.dev",
            "Port": 8883,
            "Client ID": "<generated clientId>",
            {/* under Security */}
            "Username": "<generated username>",
            "Password": "<generated password>"
          }
          ```
  </TabItem>
</Tabs>

### Probar conexiones
<Steps>
1. On the **Edge** page, make sure the status of each client under the **Clients** tab is normal.
2. Copy a UNS model topic and publish a message to it through the client.
3. Go back to **UNS**, check the data under the target topic.
</Steps>

## API
Use **RestAPI** to operate on UNS data, flows and authentication.
:::tip[Cómo usar la API]
- El base path de OpenAPI es `/openapi/v1`. Configura `TIER0_BASE_URL` con tu host Tier0 más el base path, por ejemplo `https://your-workspace.tier0.app/openapi/v1`, y pasa tu API key en el header `x-api-key`.
- Las API keys se generan en **Settings > API Keys** haciendo clic en el icono de cuenta en la esquina superior derecha.
:::

- `/flow/create`

  Crea un **Source Flow** o **Event Flow**. `flowName` y `flowType` son obligatorios.

  <Tabs syncKey="flow-api-language">
    <TabItem label="JavaScript">

      ```js
      const BASE_URL = process.env.TIER0_BASE_URL;
      const API_KEY = process.env.TIER0_API_KEY;

      const response = await fetch(`${BASE_URL}/flow/create`, {
        method: 'POST',
        headers: {
          'content-type': 'application/json',
          'x-api-key': API_KEY,
        },
        body: JSON.stringify({
          flowName: 'Manufacturing source flow',
          flowType: 'SourceFlow',
          description: 'Collects UNS source data from the plant.',
          template: '{}',
        }),
      });

      const result = await response.json();
      console.log(result.data.id, result.data.brokerID);
      ```

    </TabItem>
    <TabItem label="Python">

      ```python
      import os
      import requests

      base_url = os.environ["TIER0_BASE_URL"]
      api_key = os.environ["TIER0_API_KEY"]

      response = requests.post(
          f"{base_url}/flow/create",
          headers={"x-api-key": api_key},
          json={
              "flowName": "Manufacturing source flow",
              "flowType": "SourceFlow",
              "description": "Collects UNS source data from the plant.",
              "template": "{}",
          },
          timeout=30,
      )
      response.raise_for_status()

      result = response.json()
      print(result["data"]["id"], result["data"]["brokerID"])
      ```

    </TabItem>
    <TabItem label="cURL">

      ```bash
      curl -X POST "$TIER0_BASE_URL/flow/create" \
        -H "content-type: application/json" \
        -H "x-api-key: $TIER0_API_KEY" \
        -d '{
          "flowName": "Manufacturing source flow",
          "flowType": "SourceFlow",
          "description": "Collects UNS source data from the plant.",
          "template": "{}"
        }'
      ```

    </TabItem>
  </Tabs>

  Lista flows y opcionalmente filtra por `flowType` o `keyword`. Úsalo para encontrar el flow `id` antes de operaciones UPDATE o GET.

- `/flow/update`

  Actualiza un flow por `id`. Envía solo los campos que quieras cambiar.

  <Tabs syncKey="flow-api-language">
    <TabItem label="JavaScript">

      ```js
      const BASE_URL = process.env.TIER0_BASE_URL;
      const API_KEY = process.env.TIER0_API_KEY;
      const flowId = 123;

      const response = await fetch(`${BASE_URL}/flow/update`, {
        method: 'POST',
        headers: {
          'content-type': 'application/json',
          'x-api-key': API_KEY,
        },
        body: JSON.stringify({
          id: flowId,
          flowName: 'Manufacturing source flow v2',
          description: 'Updated source flow description.',
          isFavorite: 1,
        }),
      });

      const result = await response.json();
      console.log(result.data.success);
      ```

    </TabItem>
